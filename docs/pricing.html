Pricing & Cost Strategy

Infrastructure cost model for a 50,000-peak-CCU, five-region online-gaming platform — compute, storage, networking, analytics, monitoring, DR, and multi-region redundancy, with a total-cost-of-ownership optimization plan.

1 · Cost Summary

Two cost centers dominate: game-server compute (scales with concurrent users) and the platform (operations + data + analytics + monitoring, relatively fixed). The first is highly optimizable; the second is modest by comparison.

Headline: a naïve all-On-Demand build of this platform runs ~$132k/mo. Applying the optimization levers in Section 6 (Spot fleets, Graviton, Linux, Savings Plans, S3 tiering) brings it to ~$54k/mo — the same workload, ~59% cheaper. Game-server hosting is the lever that matters; everything else is rounding by comparison.

3 · SLA Tiers — Monitoring & Availability

Cost scales with the availability/observability guarantee. GameCloud targets Gold for a live multiplayer platform.

TierAvailabilityMonitoringArchitectureCost impact
Bronze99.0%CloudWatch basic, daily alarmsSingle-AZ, manual scalingbaseline
Silver99.9%CloudWatch + dashboards + SNS alertsMulti-AZ, auto-scaling+25%
Gold ◄99.95%+ X-Ray tracing, OpenSearch real-time, on-call pagingMulti-AZ + multi-region failover+55%
Platinum99.99%+ Shield Advanced, 24/7 SOC, synthetic canariesActive-active multi-region+140%
A live multiplayer game lives and dies on uptime during peak hours and tournaments — Gold (99.95%) is the floor. Platinum's active-active multi-region is the target only for flagship esports events, provisioned on-demand via the tournament workflow rather than run year-round.

5 · Multi-Region / Multi-City Redundancy Cost

Expanding regions buys lower player latency + outage isolation, at a predictable multiplier on the platform (not game servers, which are placed per-player anyway).

TopologyLatencyResiliencePlatform multiplierPlatform / mo (optimized)
Single regionhigh for distant playersAZ-level only1.0×$4,513
2-region active-passiveimprovedsurvives 1 region loss1.6×$7,221
3-region active-active ◄low globallysurvives 1 region, no failover lag2.5×$11,283
5-region (our seed map)lowestregional isolation everywhere3.8×$17,150
Active-passive standbys cost ~60% of a full region (warm, scaled-down) and scale up only on failover — a deliberate cost/RTO trade. Read replicas + S3 CRR + DynamoDB Global Tables make the data layer multi-region without doubling compute.
